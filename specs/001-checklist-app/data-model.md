# Data Model: Checklist App

**Phase**: 1 — Design
**Date**: 2026-05-20
**Branch**: `001-checklist-app`

---

## Entities

### User Account

Managed entirely by Supabase Auth. The application never stores user records in custom tables —
all joins to user data use `auth.uid()` from the session context.

| Attribute | Type | Notes |
|-----------|------|-------|
| id | UUID | Primary key, provided by Supabase Auth |
| email | string | Used for login |

**Relationships**: Owns zero or more Checklists.

---

### Checklist

Represents a titled, organized list generated from a user's input.

| Attribute | Type | Constraints | Notes |
|-----------|------|-------------|-------|
| id | UUID | PK, auto-generated | |
| user_id | UUID | NOT NULL, FK → auth.users | RLS filter key |
| title | text | NOT NULL | Generated by AI or provided by user |
| prompt | text | NULLABLE | Original user input (AI/extraction path only) |
| created_at | timestamptz | NOT NULL, default NOW() | Used for rate-limit queries |

**Relationships**: Belongs to one User Account. Contains zero or more Checklist Items.

**RLS Policy**: SELECT/INSERT/UPDATE/DELETE only permitted where `user_id = auth.uid()`.

---

### Checklist Item

A single task within a checklist.

| Attribute | Type | Constraints | Notes |
|-----------|------|-------------|-------|
| id | UUID | PK, auto-generated | |
| checklist_id | UUID | NOT NULL, FK → checklists.id | Cascade delete |
| user_id | UUID | NOT NULL, FK → auth.users | Redundant with checklist.user_id; required for RLS |
| text | text | NOT NULL | The task description |
| category | text | NULLABLE | Grouping label (e.g., "Setup", "Testing") |
| position | integer | NOT NULL | Display order within category |
| checked | boolean | NOT NULL, default false | Completion state |

**Relationships**: Belongs to one Checklist. Belongs to one User Account.

**RLS Policy**: SELECT/INSERT/UPDATE/DELETE only permitted where `user_id = auth.uid()`.

---

## State Transitions

### Checklist Item completion state

```
unchecked (checked = false)
       │
       │ User toggles item
       ▼
 checked (checked = true)
       │
       │ User toggles item
       ▼
unchecked (checked = false)
```

No terminal state — items can be toggled freely.

## Validation Rules

| Entity | Field | Rule |
|--------|-------|------|
| Checklist | title | Non-empty string |
| Checklist | user_id | Must match authenticated user's ID |
| Checklist Item | text | Non-empty string |
| Checklist Item | position | Non-negative integer |
| Checklist Item | checklist_id | Must reference a checklist owned by the same user |

---

## Rate Limit Logic

Rate limiting is evaluated at the Server Action layer, not enforced by a database constraint:

```
COUNT of checklists WHERE user_id = current_user
                      AND created_at >= NOW() - INTERVAL '1 hour'
                      >= 20  →  REJECT with user-facing error
```

Applies only to: `generateChecklist`, `generateFromExtraction`.
Does not apply to: `createFromSpreadsheet`, `createManualChecklist`.
